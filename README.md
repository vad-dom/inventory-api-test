## Бизнес-правила

### Товар

- Операции с остатками разрешены только для активных товаров.
- Товар нельзя деактивировать, если по нему есть положительный остаток.
- Товар нельзя удалить, если по нему есть остатки или история операций.

### Склад

- Операции с остатками разрешены только для активных складов.
- Склад нельзя деактивировать, если на нем есть положительный остаток.
- Склад нельзя удалить, если по нему есть остатки или история операций.

### Складские операции и остатки

- Каждая складская операция сохраняется в истории операций.
- Остаток не может быть отрицательным.
- Приход увеличивает остаток.
- Списание уменьшает остаток.
- Перемещение уменьшает остаток на складе-источнике и увеличивает на складе-получателе.
- При перемещении склад-источник и склад-получатель должны отличаться.
- Любое изменение остатков должно выполняться внутри транзакции.
- При одновременных операциях не должно возникать отрицательных остатков.

## Архитектура

Проект построен с разделением ответственности между слоями приложения.

- **Controllers** — принимают HTTP-запросы и возвращают HTTP-ответы.
- **Services** — содержат бизнес-логику складских операций.
- **Repositories** — инкапсулируют работу с базой данных.
- **API Resources** — отвечают за формирование JSON-ответов.
- **Form Requests** — выполняют валидацию входящих запросов.
- **Observers** — используются для централизованной инвалидации кэша статистики.

## 🚀 Как запустить проект

Клонировать репозиторий:

```bash
git clone https://github.com/vad-dom/inventory-api-test.git
cd inventory-api-test
```

Собрать и запустить контейнеры:

```bash
docker compose up -d --build
```

При запуске автоматически:

- поднимутся MySQL, Redis, PHP-FPM, Nginx и Swagger UI;
- создастся `.env` из `.env.example`;
- установятся зависимости Composer;
- сгенерируется `APP_KEY`;
- выполнятся миграций;
- заполнятся тестовые данные (Seeders).

После запуска API будет доступно по адресу:

```
http://localhost:8080/api/v1
```

---

## Переменные окружения (.env)

При первом запуске файл `.env` создается автоматически из `.env.example`.

При необходимости его можно скопировать вручную:

```bash
cp .env.example .env
```

После изменения переменных окружения рекомендуется выполнить:

```bash
docker compose exec php php artisan config:clear
```

---

## Запуск миграций

При первом запуске миграции выполняются автоматически.

При необходимости выполнить их повторно:

```bash
docker compose exec php php artisan migrate
```

или полностью пересоздать базу данных:

```bash
docker compose exec php php artisan migrate:fresh
```

---

## Запуск Seeders

При первом запуске Seeders выполняются автоматически.

При необходимости можно выполнить их вручную:

```bash
docker compose exec php php artisan db:seed
```

или вместе с полным пересозданием базы:

```bash
docker compose exec php php artisan migrate:fresh --seed
```

Seeders создают фиксированный набор данных:

- товары;
- склады;
- остатки товаров;
- историю складских операций.

Эти данные используются для проверки API и примеров в OpenAPI-документации.

```

```


Для проверки эндпойнтов в Swagger нужно нажать **Authorize** и указать API-ключ:

```

```
